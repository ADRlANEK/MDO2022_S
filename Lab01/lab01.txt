Sprawozdanie lab 01 

1. git clone <adres>
	Klonowanie repozytorium na lokalną maszyne. Polecenie zostało użyte do 
	
2. git checkout -b DL299976 
	Polecenie do zmiany brancha na którym obecnie się znajdujemy. Flaga -b pozwala utworzyć nowy branch o danej nazwie.

3. git status
	Sprawdza status plików

4. git commit -m <wiadomość>
	Tworzenie nowego commita. Flaga -m pozwala dodać wiadomość.

5. git push --set-upstream origin BB401928 
	Polecenie wypycha zmiany z lokalnego do globalnego repozytorium. Flaga --set-upstream mówi, że wypychamy na brancha, którego jeszcze nie ma globalnie.

6. git add . 
	Polecenie dodaje pliki do następnego commita. Kropka oznacza, że wybieramy wszystkie pliki.
	
7. Pull request służy do wystawienia zmian i późniejszego zmergowania ich z innym branchem. Będzie to możliwe jeśli zostanie on zaakceptowany i nie będzie żadnych kolizji z gałęzią grupy.
